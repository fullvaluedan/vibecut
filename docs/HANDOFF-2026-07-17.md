## Hard-won lessons THIS session (enforce on every subagent)

- **Agent worktrees base on origin/main, not the current branch.** Two of three round-8 agents
  built against pre-round-6 code; merges had to be reconciled by INTENT (the third agent
  fast-forwarded its worktree to the branch tip first: tell agents to do that explicitly).
  Also: .claude/worktrees/ was NOT gitignored (it is now); never git add -A while worktrees exist.
- **Never edit pipeline code while a background eval is reading the tree**: later fixtures would
  pick up different code than earlier ones.
- **The in-session browser pane can wedge at 0x0 viewport** (twice today). Fallback that works:
  claude-in-chrome on Dan's real Chrome (Browser 2), new tab. computer-tool clicks map screenshot
  space correctly, but getBoundingClientRect returns NATIVE viewport coords (2560-wide): scale by
  screenshotWidth/viewportWidth before comparing.
- **Media import for in-app tests**: file_upload only accepts session-shared paths, and Dan's real
  clips exceed its 10MB cap. Working recipe: copy a small fixture into apps/web/public/, then
  in-page fetch -> File -> DataTransfer -> dispatch DragEvent('drop') on the drop zone. The
  bin's "Assemble all assets" button places it on the timeline (synthetic mouse drags from the
  bin do NOT carry the app's drag payload). Delete the public/ copy afterwards.
- **Timeline interaction tests**: resize handles are a few px INSIDE the clip edge; verify
  outcomes numerically via window.__vibeEditor.scenes.getActiveScene().tracks, not screenshots.
- **Whisper hallucinates fluent text over silence** (30s "Thank you." segments, 9-21s words) and
  fixtures carried hundreds of such words; any transcript-trusting logic must check words against
  their own audio spans (word-SPAN energy, not whole-segment mean: the review caught that).
- The standing rules still bind: no em dashes anywhere (grep added lines for U+2014); Edit tool
  only, never scripted string replacement; PATCHES.md row in the SAME commit for upstream edits;
  eval prompt changes need version-constant bumps (retake/structural still lack them); bun
  mock.module is process-global; never claude --version in-repo; foreground evals.
